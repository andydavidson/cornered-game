# Cornered

A two-player abstract strategy game played on a configurable grid. Each player races a pawn to the opposite side of the board while placing walls to slow their opponent. Playable in the browser against an AI or in hotseat mode, installable as a PWA.

## Rules summary

- **Board**: N×N grid of cells (default 9×9, configurable 5–13, odd sizes only).
- **Start**: Player 1 at the centre of the bottom edge, Player 2 at the centre of the top edge.
- **Goal**: Be first to reach any cell on the far edge.
- **Each turn**, do exactly one of:
  1. Move your pawn one cell orthogonally (no diagonal, except the jump-around case below).
  2. Place a wall from your remaining allowance.
- **Jumping**: if the opponent's pawn is directly adjacent you may jump straight over it. If the cell beyond is off-board or wall-blocked you may instead move diagonally to a cell beside the opponent.
- **Walls** span two cell-boundaries and cannot be placed if they would cut off either player's only route to their goal (BFS check enforced before every placement).
- **Walls per player** scale with board size: `round(N × 10 / 9)` — 10 walls each on a 9×9 board.

## Repo layout

```
/packages
  /engine   — pure TypeScript game logic, zero UI/DOM deps
  /ai       — minimax + alpha-beta opponent with aggression parameter
  /modes    — LocalVsAI and LocalHotseat orchestration
/apps
  /web      — SvelteKit app: SVG board, setup screen, PWA manifest + service worker
```

## Prerequisites

- **Node.js** 20+ (LTS)
- **npm** 9+ (ships with Node 20)

## Standing up a local dev server

```bash
# 1. Clone the repo
git clone <repo-url>
cd cornered-game

# 2. Install all workspace dependencies
npm install

# 3. Start the dev server
npm run dev
```

Open [http://localhost:5173](http://localhost:5173) in your browser. The board is live-reloadable — changes to any package are picked up automatically.

## Running tests

```bash
npm test
```

Runs Vitest across `packages/engine` (31 tests) and `packages/ai` (6 tests). No browser or UI tests are required — all game logic is covered by the engine unit tests.

To run a single package in watch mode:

```bash
npm run test:watch -w packages/engine
```

## Building for production

```bash
npm run build
```

Outputs a fully static site to `apps/web/build/` (via `@sveltejs/adapter-static`). The build includes a PWA service worker generated by `vite-plugin-pwa`, enabling offline play and "Add to home screen" on mobile.

Deploy the `apps/web/build/` directory to any static host:

| Host | Command |
|---|---|
| Cloudflare Pages | Point to `apps/web/build/` in dashboard |
| Netlify | `netlify deploy --dir apps/web/build` |
| GitHub Pages | Copy `build/` contents to the Pages branch |
| Local preview | `npm run preview -w apps/web` → http://localhost:4173 |

## Configuration (pre-game setup screen)

| Setting | Options |
|---|---|
| Board size | 5×5 – 13×13 (odd values only); wall count shown live |
| Game mode | vs AI · Hotseat (two players, same device) |
| AI aggression | 0 (pure racer) → 1 (wall obsessed) |
| AI difficulty | Easy / Medium / Hard (minimax depth 1–3) |
| Turn timer | Off · 15 s · 30 s · 60 s · 120 s per turn |
| Timer timeout | Miss a turn · Forfeit the game |

## Architecture notes

`packages/engine` exports only pure functions and plain-JSON types. `GameState` is serialisable with no class instances — designed for a future WebSocket relay (networked mode) with no serialisation overhead.

The AI uses minimax + alpha-beta pruning. Wall moves at deeper search nodes are pruned to a curated set (walls that lengthen the opponent's BFS shortest path) to keep response times fast on larger boards.

The turn timer is driven entirely by the UI (`apps/web/src/lib/Timer.svelte`). The engine only receives a timeout declaration and applies the configured behaviour (`miss` or `forfeit`).
